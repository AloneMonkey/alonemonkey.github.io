密码安全

密码检测

注册的时候进行密码检测:

分为弱、中、强，弱密码不能通过注册，比如: 少于6位数或者全是数字之类的，后台也要对其强弱进行验证检测。

密码存储

  1. 不能简单的对密码进行哈希加密
  2. 加盐应该使用加密的安全随机数生成器(CSPRNG)
  3. 每个用户的每一个密码都要使用独一无二的盐值
  4. 不用告诉用户输错的账号还是密码
  5. 为了使密码难以破解，可以使用慢哈希函数

存储密码的步骤：

  1. 使用 CSPRNG 生成足够长的随机盐值。
  2. 将盐值混入密码，并使用标准的密码哈希函数进行加密，如Argon2、 bcrypt 、 scrypt 或 PBKDF2 。
  3. 将盐值和对应的哈希值一起存入用户数据库。

校验密码的步骤：

  1. 从数据库检索出用户的盐值和对应的哈希值。
  2. 将盐值混入用户输入的密码，并且使用通用的哈希函数进行加密。
  3. 比较上一步的结果，是否和数据库存储的哈希值相同。如果它们相同，则表明密码是正确的；否则，该密码错误。

使用的哈希算法

可以使用：

精心设计的密钥扩展算法如 PBKDF2 、bcrypt 和 scrypt 。
OpenWall的的 Portable PHP password hashing framework。
PBKDF2在PHP、C＃、Java和Ruby的实现。
crypt 的安全版本。

不可使用：

快速加密哈希函数，如 MD5 、SHA1、SHA256、SHA512、RipeMD、WHIRLPOOL、SHA3等。
crypt()的不安全版本。
任何自己设计的加密算法。只应该使用那些在公开领域中的、由经验丰富的密码学家完整测试过的技术。

防止暴力破解

第一个是上面说的慢哈希，另外就是如果用户在某段时间内，错误次数太多的话，可以让加入验证码拦截或者暂时冻结账户。

Docker安全

Docker的安全性主要体现在如下几点：

  1. Docker容器是否会危害到宿主机或其它容器
  2. 用户如何确保下载下来的镜像是可信的、未被篡改的
  3. 确保client和daemon的连接是可信的

可以从限制容器资源使用、日志审计、监控、文件系统防护、权限控制等方面去设置。

nodejs安全

HTTP头

  • Strict-Transport-Security：强制使用安全连接（SSL/TLS之上的HTTPS）来连接到服务器。
  • X-Frame-Options：提供对于“点击劫持”的保护。
  • X-XSS-Protection：开启大多现代浏览器内建的对于跨站脚本攻击（XSS）的过滤功能。
  • X-Content-Type-Options： 防止浏览器使用MIME-sniffing来确定响应的类型，转而使用明确的content-type来确定。
  • Content-Security-Policy：防止受到跨站脚本攻击以及其他跨站注入攻击。

可以使用nodejs的koa-helmet模块设置。

koa-helmet

敏感数据

不要在代码中暴露密钥之类的敏感数据

防止暴力破解

暴力破解即系统地列举所有可能的结果，并逐一尝试，来找到正确答案。在web应用中，用户登陆就特别适合它发挥。

通过限制用户的连接频率来防止这类的攻击。在Node.js中，你可以使用koa-ratelimit包。

Session管理

cookie属性设置:

  • secure - 这个属性告诉浏览器，仅在请求是通过HTTPS传输时，才传递cookie。
  • HttpOnly - 设置这个属性将禁止javascript脚本获取到这个cookie，这可以用来帮助防止跨站脚本攻击。

在node.js可以使用cookie-session来管理

CSRF

跨站请求伪造（CSRF）是一种迫使用户在他们已登录的web应用中，执行一个并非他们原意的操作的攻击手段。这种攻击常常用于那些会改变用户的状态的请求，通常它们并不窃取数据，因为攻击者并不能看到响应的内容。

nodejs中可以使用csrf来缓和这种攻击。

csrf

数据合法性

SQL注入:
使用参数化查询（parameterized queries）或预处理语句（prepared statements）

npm

使用Node Security project(nsp)来检查你使用的模块是否是易被一些已知的手段攻击的。
